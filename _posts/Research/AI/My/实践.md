# 数据集使用

当我手上有一堆数据的时候，如何正确使用数据？

我们可以把数据集分为以下三个部分：

- **Train Set（训练集）**：平常用于训练的数据。
- **Validation Set（验证集）**：测试训练效果的数据，根据验证集结果可以及时改变训练策略。
- **Test Set（测试集）**：最终测试，只能用一次。

但是如果直接这样分，我总是只能用一部分数据来训练模型，这在数据集较小的时候可能不能保证获得最好的效果。

## Bias–variance decomposition

$$y = f(x) + \varepsilon$$

$\mathbb E[\varepsilon] = 0$

公式：$\mathbb E[(X - a)^2] = \text{Var}(X) + (\mathbb E[X] - a)^2$

$D$ 是样本的集合，注意 $\hat f$ 是关于 $D$ 的函数。

$$\begin{aligned}
    & \mathbb E_{D, \varepsilon}[(\hat f(x) - y)^2] \\
    =& \mathbb E_D[\mathbb E_\varepsilon[(\varepsilon - (f(x) - \hat f(x)))^2]]\\
    =& \mathbb E_D[\operatorname{Var}(\varepsilon) + (\mathbb E[\varepsilon] - (f(x) - \hat f(x)))^2] \\
    =& \operatorname{Var}(\varepsilon) + \mathbb E_D[(\hat f(x) - f(x))^2] \\
    =& \operatorname{Var}(\varepsilon) + \operatorname{Var}(\hat f(x)) + (\mathbb E[\hat f(x)] - f(x))^2
\end{aligned}$$

$$\boxed{\begin{aligned}
    & \mathbb E[(\hat f(x) - y)^2] \\
    =& \operatorname{Var}(\hat f(x)) && \text{Variance} \\
    +& (\mathbb E[\hat f(x)] - f(x))^2 && \text{Bias}^2 \\
    +& \operatorname{Var}(\varepsilon) && \text{Noise}
\end{aligned}}$$

Confusion matrix

|  | 预测为正类 P | 预测为负类 N |
| - | - | - |
| 真实为正类 | $\textcolor{#03C03C}{T}P$ | $\textcolor{#E32636}{F}N$ |
| 真实为负类 | $\textcolor{#E32636}{F}P$ | $\textcolor{#03C03C}{T}N$ |

Accuracy（准确率）被定义为整个表格的正确率：

$$\text{Accuracy} = \frac {\textcolor{#03C03C}{T}P + \textcolor{#03C03C}{T}N} {TP + TN + FP + FN}$$

Precision（精确率）被定义为第一列的正确率：

$$\text{Precision} = \frac {T\textcolor{#1E90FF}{P}} {T\textcolor{#1E90FF}{P} + F\textcolor{#1E90FF}{P}}$$

Recall（召回率）被定义为第一行的正确率：

$$\text{Recall} = \frac {TP} {TP + FN}$$

F1-score 被定义为 Precision 和 Recall 的调和平均数：

$$\text{F1-score} = \frac 2 {\frac 1 {\text{Precision}} + \frac 1 {\text{Recall}}}$$

F1-score 对小值较敏感——当 Precision 和 Recall 中的任何一个偏小时，F1-score 都会偏小。

# 预处理（特征工程）

这是一个比较自由的过程，一般具体问题具体分析。下面是一些通用的技巧。

## 数据清洗

这个非常简单，一般就是处理缺失值和异常值。

- 缺失值可以用均值 / 中位数 / 众数 / 一个常数等各种统计量填充，或者是直接删去带有缺失值的数据。
- 异常值一般考虑替换为临界值或当作缺失值处理，或者干脆删除。

## 标准化 & 归一化

许多机器学习算法都是针对无量纲的数据设计的。因此，在使用算法之前应当先消除数据的量纲。

最简单的方法是标准化：

$$x \gets \frac {x - \mu} \sigma$$

若需要严格限制值域在 $[0,1]$，可以使用归一化：

$$x \gets \frac {x - x_{\min}} {x_{\max} - x_{\min}}$$

## 相关性

- 计算特征两两之间的相关性——若两个特征强相关，则可以认为冗余，删去一个。
- 计算每个特征与标签之间的相关性——若特征和标签弱相关，则可以认为没用，删去。

常用的计算相关性的方法：
- 连续 & 连续：Pearson 相关系数 / Spearman 相关系数 / 互信息
- 连续 & 离散：ANOVA
- 离散 & 离散：卡方检验 / Cramer's V

## PCA

将高维空间的数据投影到低维空间（减少特征）。

如何选择低维空间的基？选择原数据方差最大的几个正交方向，或者说使得转动惯量最小的几个正交的轴。具体计算不展开。

# 过拟合的处理方法

增加数据永远是好方法。但是如果没那么多数据呢？

## Weight Decay

可以认为这是一个工程 trick，并没有多深刻的数学原理。

参数过大往往意味着过拟合的发生。因此我们根据参数的大小设置一个惩罚项 $\operatorname{Penalty}$，乘上一个量纲转换的超参数 $\lambda$ 后直接加到 loss function 上：

$$\boxed{
    L(\theta) = L_{\text{original}}(\theta) + \lambda \cdot \operatorname{Penalty}(\theta)
}$$

常见的惩罚项选择有两种，分别称为 $L_1$ Regularization 与 $L_2$ Regularization。

$L_1$ Regularization：

$$\operatorname{Penalty}(\theta) := \sum_i \lvert \theta_i \rvert$$

$L_2$ Regularization：

$$\operatorname{Penalty}(\theta) := \sum_i \theta_i^2$$

有人认为，$L_1$ Regularization 的等高线非常尖锐，因此更容易产生稀疏解。

对于线性回归（最小二乘法），它搭配上这两种 regularization 则有两个 fancy 的名字：
- 若搭配上 $L_1$ Regularization 则称为 LASSO Regression；
- 若搭配上 $L_2$ Regularization 则称为 Ridge Regression。

## Early Stopping

连续 $n$ 次进步 $< \varepsilon$ 则结束训练。

## Dropout

原论文：[Dropout: A Simple Way to Prevent Neural Networks from Overfitting](https://www.jmlr.org/papers/volume15/srivastava14a/srivastava14a.pdf?utm_content=buffer79b43&utm_medium=social&utm_source=twitter.com&utm_campaign=buffer,)

Dropout 层对前一层的输出进行选择性地遮盖（掩码）：

- 训练时，每一个输出有 $p$ 的概率变成 $0$。
  - 当然结果需要相应缩放到 $\frac 1 {1-p}$ 倍，以保证期望不变。
- 测试时不更改任何输出。

由于该层在训练时与测试时的表现不一致，切记在代码中要切换模型的模式！

注意：dropout 只能保证期望不变，方差是会变化的。对方差敏感的回归问题中不建议使用 dropout 层！[为什么回归问题不能用Dropout - 知乎](https://zhuanlan.zhihu.com/p/561124500)；[Pitfalls with Dropout and BatchNorm in regression problems | Towards Data Science](https://towardsdatascience.com/pitfalls-with-dropout-and-batchnorm-in-regression-problems-39e02ce08e4d/)

# ML 领域黑话

- **Baseline**：一个用于参照的模型。你的模型只有跑赢 baseline 才有意义。
- **Benchmark**：公认的、标准化的测试集。各类模型都在同一个 benchmark 测试才有可比性。
- **SOTA (State-of-the-Art)**：字面意思是一门工艺当前的发展水平，在 ML 中指目前最佳水平的模型。
- **CV (Cross Validation)**：自测时在验证集得到的分数。
- **LB (Leaderboard)**：排行榜，即提交后在测试集得到的分数。
- **Ground truth**：代表问题真实答案的标准数据。
- **Cherry-pick**：择优展示，含贬义。只挑运气好的实验放进报告而故意忽略表现差的结果，使模型看起来性能很漂亮实则不可复现。
- **Vanilla**：原意为香草，在英语中引申为“普通的、基本的”。在 ML 中特指最标准、最朴素、没有额外修饰或改进的版本。例如 vanilla gradient descent 指最基本的没有魔改的梯度下降法。
