# PyTorch

## `tensor`

### 基础运算

```py
a = torch.zeros((height, width)) # 全 0 tensor
b = torch.ones((height, width))  # 全 1 tensor
c = torch.randn((height, width)) # 每个元素符合标准正态分布的 tensor
```

```py
a = torch.tensor([1, 2, 3])
b = torch.tensor([4, 5, 6])
print(a + b) # tensor([5, 7, 9])
print(a * b) # tensor([ 4, 10, 18])
```

```py
a = torch.tensor([1, 2, 4])
print(torch.sum(a))          # tensor(7)
print(torch.max(a))          # tensor(4)
print(torch.mean(a.float())) # tensor(2.3333)
```

### 形状

`reshape` 和 `view` 可以改变 `tensor` 的形状，效果一致：

```py
import torch

a = torch.tensor([1, 2, 3, 4, 5, 6])
print(a.reshape(2, 3))
print(a.view(2, 3))

"""
tensor([[1, 2, 3],
        [4, 5, 6]])
"""
```

若传入的大小中有一维为 `-1`，则意味着这一维的大小会被自动推断（如果遇到不能整除的情况会报错）：

```py
print(a.reshape(-1, 3)) # 输出同上
```

`reshape` 与 `view` 区别在于 `view` 要求内存连续，因此建议无脑使用 `reshape`。

## Autograd

自动求导：

```py
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2
y.backward()

print(f"x: {x.item()}, y: {y.item()}, dy/dx: {x.grad.item()}")
# x: 3.0, y: 9.0, dy/dx: 6.0
```

梯度累加与清空：

```py
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2
z = x ** 3

y.backward()
print(f"dy/dx at x=3: {x.grad}")
# dy/dx at x=3: 6.0

z.backward()
print(f"dz/dx at x=3 (?): {x.grad}")
# dz/dx at x=3 (?): 33.0

x.grad.zero_()  # Reset gradients
z = x ** 3
z.backward()
print(f"dz/dx at x=3: {x.grad}")
# dz/dx at x=3: 27.0
```

使用 `torch.no_grad()` 禁止梯度计算，节省性能：

```py
x = torch.tensor(3.0, requires_grad=True)

y = x ** 2
y.backward()
print(f"dy/dx at x=3: {x.grad}")
# dy/dx at x=3: 6.0
print(f"y requires_grad: {y.requires_grad}")
# y requires_grad: True
x.grad.zero_()

with torch.no_grad():
    z = x ** 3
    print(f"z requires_grad: {z.requires_grad}")
    # z requires_grad: False
    # You cannot z.backward()
```

## `optim`

虽然 `optim` 提供了很多优化器，但是一般无脑用 `AdamW`。

```py
LEARNING_RATE = 1e-3 # Example
WEIGHT_DECAY = 1e-2  # Example

optimizer = optim.AdamW(model.parameters(), lr=LEARNING_RATE, weight_decay=WEIGHT_DECAY)
```

learning rate 也不是一成不变的，我们可以使用 scheduler 来随着 epoch 调整 learning rate。一般无脑用 `lr_scheduler.CosineAnnealingLR`。

```py
scheduler = CosineAnnealingLR(optimizer, T_max=EPOCHS)

for epoch in range(EPOCHS):
    model.train()

    for inputs, labels in train_loader:
        inputs, labels = inputs.to(device), labels.to(device)

        optimizer.zero_grad()
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

    scheduler.step()
```

可以通过 `scheduler.get_last_lr()[0]` 获取当前的 learning rate。
